What's new in Python 3 (aka Python 3000)

Introduction


Python 3 should be released by October 2008[1]. It is not expected that everybody will migrate their programs just after version 3 is released, because most third party libraries that most programs depends on, won't be ready for that time. So don't bother to start coding for Python 3 at least up to when version 3.1 is ready. Although Python 2.x still have a long live ahead, it won't hurt to know what we can expect from the next big release of our favorite programming language.



Datatypes


  • All strings are now Unicode.
  • Remove distinction between int and long types (L suffix is gone in longs).
  • True division is standard (that is, 5/2 now gives 2.5 and not 2 as before)
  • dict.has_key(x) is invalid, use "x in dict"
  • dict.items(), dict.keys() and dict.values() now returns an iterator instead of a list (use list() to get a list).
  • A set can be declared with {1,2,3}  instead of set([1,2,3])

Functions


  • There are function anotations: def fn(x: int) -> str: (see [2] for more information)
  • apply(f,args,kw) is now f(*args, **kw)
  • print is now a function[3]: print(1,2,3, sep="",end="\n",file=sys.stdout). This allows the programmer to change the output to a file instead of printing to the screen.

Exceptions


raise Exception, "a message" is now: exception("a message")

Removed


  • xrange() is replaced with a new range() function, the new range() will return an iterator instead of a list.
  • raw_input() is replaced with a new input() function that does what the old raw_input() did.To use the old input(), do eval(input())

Compatibility with previous versions


Python 3.0 will break backwards compatibility with Python 2.x. There is no requirement that Python 2.6 code will run unmodified on Python 3.0.[4] Here is the proposed path to migrate your 2.x programs to 3.0 by using Python 2.6:


  1. Port your project to Python 2.6.
  2. Turn on the Py3k warnings mode (with "-3" at command line).
  3. Test and edit until no warnings remain.
  4. Use the 2to3 tool to convert this source code to 3.0 syntax. Do not manually edit the output!
  5. Test the converted source code under 3.0.
  6. If problems are found, make corrections to the 2.6 version of the source code and go back to step 3.
  7. When it's time to release, release separate 2.6 and 3.0 tarballs (or whatever archive form you use for releases).
